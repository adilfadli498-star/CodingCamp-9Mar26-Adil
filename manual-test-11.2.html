Manual Test Guide - Task 11.2

Purpose: Verify ARIA labels and semantic HTML implementation

Test 1: ARIA Labels on Timer Buttons

Steps:

  1. Open index.html in your browser
  2. Open Developer Tools (F12)
  3. Navigate to the Elements/Inspector tab
  4. Find the timer buttons (Start, Stop, Reset)
  5. Verify each button has an aria-label attribute
Expected Result:
  • Start button: aria-label="Start timer"
  • Stop button: aria-label="Stop timer"
  • Reset button: aria-label="Reset timer"

Test 2: ARIA Live Regions for Timer

Steps:

  1. In Developer Tools, find the .timer-display element
  2. Verify it has aria-live="polite"
  3. Verify it has aria-atomic="true"
  4. Find the .timer-status element
  5. Verify it has aria-live="polite"
Expected Result:
  • Timer display: aria-live="polite" aria-atomic="true"
  • Timer status: aria-live="polite"
Note: These attributes allow screen readers to announce timer updates without interrupting the user.

Test 3: ARIA Checked for Task Checkboxes

Steps:

  1. In the todo list, add a new task (e.g., "Test Task")
  2. In Developer Tools, find the checkbox element (.todo-checkbox)
  3. Verify it has aria-checked="false"
  4. Click the checkbox to mark the task as complete
  5. Verify the attribute changes to aria-checked="true"
  6. Click again to uncheck
  7. Verify it changes back to aria-checked="false"
Expected Result:
  • Unchecked task: aria-checked="false"
  • Checked task: aria-checked="true"
  • Attribute updates dynamically when toggled

Test 4: Semantic HTML Elements

Steps:

  1. Open Developer Tools Console
  2. Run the following commands to verify semantic HTML:
// Check for button elements
document.querySelectorAll('button').length
// Expected: 6 or more

// Check for form elements
document.querySelectorAll('form').length
// Expected: 2

// Check for input elements
document.querySelectorAll('input').length
// Expected: 3 or more

// Check for ul elements
document.querySelectorAll('ul').length
// Expected: 2

// Check for semantic structure
document.querySelector('main') !== null
// Expected: true

document.querySelector('header') !== null
// Expected: true

document.querySelectorAll('section').length
// Expected: 4

Test 5: Dynamic Button ARIA Labels

Steps:

  1. Add a task with text "My Important Task"
  2. In Developer Tools, find the Edit button for this task
  3. Verify it has aria-label="Edit task "My Important Task""
  4. Find the Delete button
  5. Verify it has aria-label="Delete task "My Important Task""
Expected Result:
  • Edit button includes task text in aria-label
  • Delete button includes task text in aria-label
  • Labels are descriptive and context-specific

Test 6: Form Input ARIA Labels

Steps:

  1. In Developer Tools, find the todo input field
  2. Verify it has aria-label="New task description"
  3. Find the link name input
  4. Verify it has aria-label="Link name"
  5. Find the link URL input
  6. Verify it has aria-label="Link URL"
Expected Result:
  • All form inputs have descriptive aria-label attributes
  • Labels clearly indicate the purpose of each input

Test 7: Screen Reader Testing (Optional)

Steps:

  1. Enable a screen reader:
    • Windows: NVDA (free) or JAWS
    • macOS: VoiceOver (Cmd+F5)
    • Chrome: ChromeVox extension
  2. Navigate through the page using Tab key
  3. Verify screen reader announces:
    • Button labels (Start timer, Stop timer, etc.)
    • Input labels (New task description, etc.)
    • Checkbox states (checked/unchecked)
  4. Start the timer and verify time updates are announced
  5. Add a task and verify checkbox state is announced
Expected Result:
  • All interactive elements are announced with descriptive labels
  • Timer updates are announced politely (not interrupting)
  • Task completion state changes are announced
Note: This test requires a screen reader. If you don't have one installed, you can skip this test or install NVDA (free for Windows) or use VoiceOver (built into macOS).

Checklist - All Tests

  • Timer buttons have aria-label attributes
  • Timer display has aria-live="polite"
  • Timer status has aria-live="polite"
  • Task checkboxes have aria-checked attribute
  • aria-checked updates when task is toggled
  • Edit buttons have descriptive aria-label
  • Delete buttons have descriptive aria-label
  • Form inputs have aria-label attributes
  • Submit buttons have aria-label attributes
  • Uses semantic <button> elements
  • Uses semantic <form> elements
  • Uses semantic <input> elements
  • Uses semantic <ul> and <li> elements
  • Uses semantic structural elements (main, header, section)
  • Screen reader announces all labels correctly (optional)
